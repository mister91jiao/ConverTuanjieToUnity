# 团结工程转 Unity 工程工具使用教程

本文档目录用于记录团结工程迁移到 Unity 工程时的工具用法、安装方式、Unity 打开流程、重新打包注意事项和 Android 设置变化。

工具 Git 地址：

```text
https://github.com/mister91jiao/ConverTuanjieToUnity.git
```

Unity 菜单路径：

```text
Tools/团结转Unity/转换GUID并修复资源
```

## 文档列表

- [01-工具做了哪些工作.md](01-工具做了哪些工作.md)：说明转换工具会处理哪些内容，包括 `.meta` GUID、`AssetDatabase` children 关系和 `.scene -> .unity` 场景后缀转换。
- [02-PackageManager安装与版本管理.md](02-PackageManager安装与版本管理.md)：说明如何在团结编辑器内通过 Package Manager 添加转换工具。
- [03-转换完成后用Unity打开流程.md](03-转换完成后用Unity打开流程.md)：说明转换工具执行完成后，第一次用 Unity 打开工程时如何处理打开提示和错误 Package。
- [04-重新打包遇到的问题.md](04-重新打包遇到的问题.md)：说明迁移后重新打 AssetBundle 时需要关注的场景路径、LightingData、Lightmap 和 ReflectionProbe 问题。
- [05-Android设置以及变化.md](05-Android设置以及变化.md)：说明 AndroidManifest 和 Gradle 模板中需要从团结名称切回 Unity 名称的设置。

## 整体流程大纲

1. 在团结工程中完成最后一次可用打包或备份。
2. 仍然使用团结打开工程副本。
3. 在团结的 Package Manager 中添加转换工具包。
4. 在团结中执行 `Tools/团结转Unity/转换GUID并修复资源`。
5. 使用 Unity 打开已经转换完成的工程。
6. 处理 Unity 打开时的版本提示、Package 错误和资源导入问题。
7. 重新烘焙需要静态光照数据的场景。
8. 重新打 AssetBundle。
9. 检查 Android 配置变化，重新构建并验证设备运行。
